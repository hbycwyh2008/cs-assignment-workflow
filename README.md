# CS GitHub planning guide (CS1 / CS2)

Course materials for **`planning.md`**: policy, student-facing expectations, a 4-point planning rubric, **optional** rubrics for commits and reflection, and examples for **web**, **programming**, and **VEX Robotics** (see [Part K](CS_GITHUB_PLANNING_GUIDE.md#part-k) in the full guide).

## Start here

**End-to-end process (one page):** open **[assignment_workflow.md](assignment_workflow.md)** (plan → commits → submission → reflection).

**Everything in one file:** **[CS_GITHUB_PLANNING_GUIDE.md](CS_GITHUB_PLANNING_GUIDE.md)** — policy, student text, rubrics, and examples. The same workflow also appears at the top as **Overview** ([`#overview`](CS_GITHUB_PLANNING_GUIDE.md#overview)). **Jump to sections** links use anchors like `#part-a` (they work on github.com).

## Use with assignments

- **Download:** green **Code** → **Download ZIP**, or clone this repo.
- **Template (optional):** In the GitHub repo settings, enable **Template repository** so students can use **Use this template** for each new assignment.

Students should copy the structure into their own assignment repo as **`planning.md`** and replace the task-specific text with the current assignment.
