# Task 2 — Full-length MCQ wrong-question collection

**Scope:** Barron + *5 Steps to a 5* — full practice tests only.

**Organization:** By **topic / question type** (CSA concept), not by book. When you log a mistake, put the note under the folder that matches the **skill the question tested** (for example `arraylist/`, `inheritance/`). In each entry, still record **source** (Barron vs 5 Steps, which practice test, question number) so you can find it again.

**Suggested file naming:** `barron-test2-q15.md` or `5steps-practice4-q07.md` — the folder is the topic; the filename carries the book and position.

## Topic folders (align entries with the main idea of the question)

| Folder | Typical content |
|--------|-----------------|
| `primitive-types` | `int`, `double`, casting, arithmetic, `Math` |
| `using-objects-strings` | `String` methods, `Scanner`, API of given classes |
| `conditionals-and-logic` | `if` / `else`, boolean expressions, De Morgan, short-circuit |
| `iteration` | `for`, `while`, nested loops, tracing loop output |
| `writing-classes` | constructors, `private` fields, methods, `static`, `this` |
| `arrays` | 1D arrays, `length`, references, common algorithms |
| `arraylist` | `ArrayList` API, generics, add/remove, traversals |
| `two-dimensional-arrays` | row/column indexing, ragged arrays, 2D traversals |
| `inheritance-and-polymorphism` | subclasses, `super`, overriding, abstract classes, interfaces |
| `recursion` | base case, recursive calls, call stack intuition |
| `other-or-mixed` | questions that truly span topics or do not fit above |

If one question could fit two folders, pick the **main** skill you missed and note the secondary tag inside the file.
